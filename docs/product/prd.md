# PRD — cems

이 문서는 확정 스펙이 아니라 현재 작업을 진행하기 위한 제품 골격이다. 대회 계획서 세부와 데이터가 확인되면 짧게 갱신한다.

## 목표

건물/캠퍼스 에너지 사용량을 예측하고, 사용량이 높게 예측된 이유와 줄일 수 있는 행동을 사용자가 이해할 수 있게 보여준다.

## 1차 사용자

아직 미정이다. 당장은 다음 후보 중 하나를 골라야 한다.

| 후보 | 필요한 이유 |
| --- | --- |
| 시설/에너지 관리자 | 예측, 원인 분석, 조치 판단이 가장 직접적이다. |
| 캠퍼스 운영 담당자 | 비용/탄소 리포트와 절감 목표 관리가 중요하다. |
| 대회 심사위원 | 데모에서 기술 흐름과 효과 근거를 빠르게 봐야 한다. |

## MVP 흐름

1. 사용자가 에너지 데이터 범위 또는 샘플 건물을 선택한다.
2. 앱이 사용량 예측치를 P50 중심으로 보여주고 P10/P90 범위를 함께 표시한다.
3. 사용자가 특정 시간대나 구간을 선택하면 SHAP 기반 주요 요인을 본다.
4. LLM이 예측값과 요인 설명을 바탕으로 절감 코칭을 생성한다.
5. 앱이 예상 절감량을 kWh, 비용, tCO2 후보 단위로 요약한다.

이 흐름은 화면 설계가 아니라 제품 흐름이다. 실제 화면 수는 구현 때 줄이거나 합칠 수 있다.

## MVP 기능

| 우선순위 | 기능 | 상태 |
| --- | --- | --- |
| Must | 에너지 사용량 예측 표시 | 데이터/모델 미정 |
| Must | P10/P50/P90 예측 구간 표시 | 모델 설계 미정 |
| Must | SHAP 요인 설명 | ML 구현 미정 |
| Must | LLM 절감 코칭 | 모델/프롬프트 미정 |
| Should | kWh를 비용/tCO2로 환산 | 계수 출처 확인 필요 |
| Later | 목표 설정, 알림, 리포트, 리더보드 | MVP 이후 |

## 데이터 요구

최소로 필요한 데이터는 다음과 같다.

| 항목 | 상태 |
| --- | --- |
| 사용량 시계열 | `timestamp`, `kWh` 필요. 소스 미정. |
| 건물/공간 식별자 | 멀티 건물 여부 미정. |
| 외생 변수 | 날씨, 요일, 운영 일정 후보. 실제 사용 여부 미정. |
| 기준선 | 절감률 계산 기준. 정의 미정. |

## 성공 기준 후보

- 모델: MAPE 또는 MAE, P10/P90 커버리지, Pinball loss.
- 제품: 코칭이 예측/SHAP 근거를 반영하는지, 사용자가 다음 행동을 이해하는지.
- 대회: 원본 계획서의 채점 기준 확인 필요.
